Clicker Game

**Platform:** MIT App Inventor

This project demonstrates event-driven programming, game loop logic, and visual UI design principles using block-based coding. In this game a baseball is trying to get across the field for a home run and the player must click on it to get points. Certain things the player does may lose them points and it is all about letting in the least amounts of runs possible.

Detailed Project Breakdown:

This app was made using a block-based coding technique on MIT App Inventor to make a clicker game where the user had to click a sprite (the baseball) as many times as possible in 20 seconds. The ball would go from left to right and when it was clicked it would go to a random spot on the left side of the page and travel across and if it reached the end the user would lose a point. The user would also lose a poitn if they missed the ball when clicking. When the ball is clicked th esprite will change to an image of the ball in a glove to add a sort of "animation" feature as an added design choice. Randomly throughout the game the ball will speed up for a short period of time and then possibly return. After the game ends the user is asked if they want to keep playing using a notifer to display their score. The user may also quit in the middle of the game if they click the button and the notifier. It is a fun game that incorporates multiple fun features to enhance game expierience.
